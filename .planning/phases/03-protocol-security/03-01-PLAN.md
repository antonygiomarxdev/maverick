Add comprehensive FCnt tests to `lorawan_10x_class_a.rs` `#[cfg(test)]` module.

**The existing tests cover basic cases. Add edge cases:**

1. `fcnt_wrap_65535_to_0_rollover` — session=65535 (0xFFFF), wire=0. Must reconstruct to 65536 (0x10000). This is the actual 16-bit rollover event.

2. `fcnt_wrap_65534_to_1_rollover` — session=65534 (0xFFFE), wire=1. Low wire is higher but session is near 16-bit boundary. Must NOT treat as duplicate.

3. `fcnt_retransmit_vs_rollover` — When wire FCnt wraps, how does the algorithm distinguish a retransmit (same frame re-sent) from a legitimate rollover? The current gap-based approach (D-02: if gap greater than 1000, treat as rollover). Add a test that wires at 1 with session at 65536 (gap = 65535 greater than 1000, so rollover to 65537).

4. `fcnt_max_u32_minus_one` — session=0xFFFF_FFFE (u32::MAX - 1), wire=0xFFFF. Wire wraps to rollover candidate at 0x1_0000_FFFF but that is below session (gap would be huge). Test the algorithm behavior.

5. `fcnt_zero_to_one_happy` — session=0, wire=1. Forward progress, no rollover. Should accept.

6. `fcnt_gap_1000_exactly` — D-02 says gap greater than 1000 triggers rollover detection. Test at exactly gap=1000 (should NOT assume rollover) and gap=1001 (should assume rollover).

7. `fcnt_very_large_gap_in_past` — session=100000, wire=50000 (gap=50000 greater than MAX_FCNT_GAP=16384). Should reject with GapExceeded.

**Note:** The `extend_fcnt` function already exists in `lorawan_10x_class_a.rs`. Add these as `#[test]` functions in the existing `mod tests`.
