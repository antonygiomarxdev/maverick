**Pipeline summary:**
1. UDP adapter `parse_lorawan_payload()` in gwmp.rs extracts: DevAddr, FCnt, FPort, payload, wire_mic, phy_without_mic
2. `IngestUplink::execute()` does: FCnt check to MIC verify to decrypt to dedup to persist
3. SQLite stores in uplinks table with payload_decrypted column

**UDP adapter already does (verified in research):**
- `wire_mic = raw[raw.len()-4..]` — correct per LoRaWAN spec
- `phy_without_mic = raw[..raw.len()-4]` — correct (excludes MIC)

**Existing test in gwmp.rs:**
`parse_push_data_json_into_observation` — parses base64-encoded frame and checks DevAddr, FCnt, FPort. Does NOT verify wire_mic or phy_without_mic are correctly extracted.

**Test 1: `full_pipeline_valid_frame_with_mic`**
Create a real LoRaWAN uplink frame (construct manually with known MHDR, DevAddr, FCnt, FOpts, FPort, payload, MIC) in base64. Parse it through `parse_push_data_json`. Assert:
- `dev_addr` matches the encoded DevAddr
- `f_cnt` matches the encoded 16-bit wire FCnt
- `wire_mic` matches the last 4 bytes of the raw PHY payload
- `phy_without_mic` has length = raw_len - 4

Construct the frame bytes directly to have known values:
```rust
// Manual LoRaWAN 1.0.x uplink frame construction
// MHDR (1 byte): MType=0b_000_XX_XXX (unconfirmed uplink), RFU, Major
let mhdr = 0x40u8; // unconfirmed uplink, major=00
let dev_addr_bytes = 0x04_03_02_01u32.to_le_bytes(); // DevAddr = 0x01020304
let fctrl = 0x00u8; // no FOpts, ADR=0, ACK=0, RFU=0
let fcnt_bytes = 0x00_01u16.to_le_bytes(); // FCnt = 1
let fport = 0x01u8; // FPort = 1
let frm_payload = vec![0xAA, 0xBB, 0xCC]; // 3-byte payload
// Construct raw frame for MIC: MHDR || FHDR || FPort || FRMPayload (no MIC for now)
// Then add MIC bytes computed from the known-good test vector
```

**Test 2: `mic_extraction_from_known_frame`**
Hardcode a raw PHY frame (as bytes) where the MIC is known. Parse it and verify `wire_mic` equals the exact bytes. Also verify `phy_without_mic` excludes those 4 bytes.

**Test 3: `phy_without_mic_correct_length`**
Verify `phy_without_mic.len() == raw.len() - 4` for a frame with MIC. This catches off-by-one errors in parsing.
