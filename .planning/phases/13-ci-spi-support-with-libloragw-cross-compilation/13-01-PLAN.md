## Plan: CI SPI Support with libloragw Cross-Compilation

### Goal
Make release CI build binaries WITH SPI support so auto-update works for SPI hardware devices.

### Context
libloragw sources are already vendored in `crates/maverick-adapter-radio-spi/libloragw/`. The build fails on ARM cross-compilation because build.rs doesn't pass `--sysroot` to the ARM gcc. release.yml already sets up cross-toolchains but doesn't build with SPI feature.

### Tasks

#### Task 1: Modify build.rs for cross-compilation sysroot
<read_first>
- crates/maverick-adapter-radio-spi/build.rs
- .github/workflows/release.yml (lines 115-143 for CFLAGS pattern)
</read_first>

<acceptance_criteria>
- build.rs detects CFLAGS_* environment variables for ARM targets
- build.rs parses --sysroot= from CFLAGS and passes to cc::Build
- build.rs compiles correctly for native x86_64 (no sysroot needed)
- build.rs compiles correctly for aarch64 cross-compilation (with sysroot)
- build.rs compiles correctly for armv7 cross-compilation (with sysroot)
- `cargo check -p maverick-adapter-radio-spi` passes
- `cargo check -p maverick-runtime-edge --features spi` passes
</acceptance_criteria>

<action>
Added cross-compilation detection to build.rs:
1. Reads CFLAGS_* env vars and extracts --sysroot= flags
2. Also checks CARGO_BUILD_TARGET to detect ARM targets and look for SYSROOT env vars
3. Passes --sysroot= flags to cc::Build

The build.rs now has three detection mechanisms:
- CFLAGS env var (general)
- CFLAGS_aarch64_unknown_linux_gnu and CFLAGS_armv7_unknown_linux_gnueabihf (release.yml sets these)
- CARGO_BUILD_TARGET + SYSROOT_AARCH64/SYSROOT_ARMV7 env vars (fallback)

Verified: `cargo check -p maverick-adapter-radio-spi` passes with no errors.
</action>
